# GHOST LENS（ゴーストレンズ）〜心霊写真館〜

廃洋館の中央からスマートフォンを構え、音と画面ノイズを頼りに360°の暗がりを探る心霊写真スコアアタックです。怪異を中央に0.6秒捉えてフォーカスを合わせ、90秒間で除霊スコアを競います。

## 起動

`index.html` を直接ブラウザで開くか、リポジトリルートから次を実行して `http://localhost:8137/` を開きます。

```powershell
python -m http.server 8137 --directory games/ghostlens
```

iOS 13以降でジャイロを使う場合はHTTPS環境が必要です。タイトル画面の「ジャイロで構える」は必ずユーザー操作から姿勢センサーの許可を要求し、最初の姿勢を正面としてキャリブレーションします。

## 遊び方

- ヘッドホン推奨。囁き・呻き・衣擦れの方向、EMFメーター、画面端のVHSノイズから怪異の方角を探します。
- 怪異を中央のレティクルへ入れ、フォーカスリングが0.6秒で収束して `FOCUS LOCK` になったら撮影します。
- 中心2°以内は `PERFECT ×2.0`、4°以内は `GOOD ×1.5`、それ以外は `HIT`。素早さとノーミスコンボがさらに加算されます。
- 撮影成功で残り時間が3秒増えます。フォーカス前の撮影は0点でフィルムだけ消費します。
- フィルムは12枚。使い切ると2秒の現像後に自動装填されます。
- 撮影失敗または怪異の消失でコンボが切れます。
- 浮遊霊は基本100点。這い寄りは近いほど最大300点ですが、到達されると5秒間視界を奪います。金色の残光は500点で3秒しか現れません。
- タイムアップ後は、実際に撮影したWebGLフレームがポラロイド風アルバムとして表示されます。

## 操作

### スマートフォン

- `ジャイロで構える`: DeviceOrientationを許可し、端末の向きで360°を見回す
- 画面ドラッグ: ジャイロを使わない場合の視点移動（画面幅およそ180°）
- 画面タップ / 右下の丸ボタン: 撮影
- `NEUTRAL`: 現在の端末方位を正面へ再設定
- `1.0× / 1.75×`: FOV 70° / 40°を切り替え

### PC

- マウス移動またはドラッグ: 視点移動
- クリック / `Space`: 撮影
- `WASD` / 矢印キー: 視点移動
- バッククォート `` ` ``: デバッグオーバーレイ

`SOUND` でミュートを切り替えられます。

## デバッグ

URLへ `?debug=1` を付けると内部状態を表示します。`?seed=N` で乱数を固定できます。すべてのAPIは `window.__game` から呼び出せます。

| API | 内容 |
|---|---|
| `getState()` | スコア、残り時間、カメラyaw/pitch、怪異、フォーカス、フィルム、コンボ、写真、入力・音声・描画状態を返す |
| `dump()` | 現在のシーンをテキストで返す |
| `step(ms)` | 固定刻みで決定論的に時間を進め、描画・音声も1回更新する |
| `setCamera(yawDeg, pitchDeg)` | カメラを指定方位・仰角へ直接向ける |
| `aimAtGhost(id?)` | 指定怪異、または角度が最も近い怪異へカメラを向ける |
| `spawnGhost(type, yawDeg, pitchDeg)` | `drifter` / `crawler` / `gold` を任意位置へ出現させ、IDを返す |
| `shutter()` | シャッターを切り、成否・得点内訳を返す |
| `setTime(sec)` | 残り時間を秒で設定する |
| `reloadFilm()` | フィルムを即座に12枚へ再装填する |
| `setFilm(n)` | フィルム残数を0〜12へ設定する。0なら現像を開始する |
| `restart()` | リザルトを閉じて新しいゲームを開始する |
| `unlockAudio()` | AudioContextを生成・再開する |
| `testAudio()` | シャッター、フォーカス、現像、3種の除霊音を通す |
| `testOrientation(a,b,g,calibrate)` | 合成DeviceOrientationイベントを送る。`calibrate=true` でその姿勢を正面化 |
| `setDebug(visible)` | デバッグオーバーレイを切り替える |

`window.__renderOnce(dt)` は非表示タブや停止中のrAFに依存せず、指定時間相当の描画を1回行い、レンダラー情報を返します。

### 例

```js
const id = __game.spawnGhost('drifter', 120, 10);
__game.aimAtGhost(id);
__game.step(650);
__game.shutter();
__game.getState().photos.length;
```

## 実装メモ

- Three.js r147 UMDをローカルから読み込み、`WebGLRenderer` は `preserveDrawingBuffer: true`、pixel ratio上限1.5で生成しています。
- ゲーム判定はDOM/Three.js非依存の固定刻みステートマシンです。
- 姿勢入力はDeviceOrientationのZ-X-Y intrinsicクォータニオンを構築し、キャリブレーション姿勢との相対回転から画面法線の方位・仰角を取り出します。
- 写真は撮影瞬間の保持済みWebGL canvasを縮小canvasへ描画し、ビネット・日付・鑑定結果を焼き込んだJPEG Data URLとして試合中のみ保持します。
