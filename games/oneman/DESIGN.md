# ワンマン運転士 ONE-MAN DRIVER — 設計書

3Dの鉄道運転ゲーム（電車でGO!系のオリジナル）。プレイヤーの操作は**ブレーキのみ**に集中させる。駅間は自動運転でシネマティックに車両を映し、駅接近で運転席に戻り、停止直前は横アングルで停止線との勝負をダイナミックに見せる。

## 1. ゲーム全体の状態機械

1ラン = 1路線5駅（4区間）。夕方に向かって時間が進む（区間1=昼 → 区間4=夕焼け）。

```
TITLE
  └(スタート)→ RUN_INTRO      … 路線図と行路表示（3s、スキップ可）
      └→ ┌────────── 区間ループ (i = 1..4) ──────────┐
          │ DEPART    … ドア閉→発車ベル→自動加速（~4s、運転席視点）│
          │ CRUISE    … 自動運転・シネマティックカメラ（~12-20s、タップでスキップ可）│
          │ APPROACH  … 残600mで運転席へ復帰、ブレーキ操作解禁      │
          │ FINAL     … 残15m以内かつ25km/h未満で横アングルへ        │
          │ STOPPED   … v=0 で確定 → STATION_RESULT                 │
          │   ├ 停止線通過+10m → OVERRUN（強制EB演出→減点確定）      │
          │   └ 手前1m超で停止 → CREEP（自動微速前進で再挑戦、減点）  │
          │ STATION_RESULT … 誤差cm・ランク・加点演出（~4s）          │
          │ DOORS     … ドア開→チャイム→閉（~3s）→ 次区間へ          │
          └──────────────────────────────────────┘
      └(4駅目終了)→ FINAL_RESULT … 総合スコア・称号 → TITLE
PAUSE … どの状態からも出入り（Esc/P）。CRUISE/DEPART中は入力不要
```

- 状態はgame.jsの単一enumで管理し、`getState().phase` で常に観測可能
- 各状態の遷移条件は**距離と速度のみ**（時間依存はDEPART/RESULT等の演出のみ）で決定論を保つ

## 2. カメラシステムと画面遷移

カメラは状態機械と独立した CameraDirector が管理。「ショット」の列挙と遷移規則:

| ショット | 内容 | 使う状態 |
|---|---|---|
| COCKPIT | 運転席。前面窓越しの一人称、計器HUD表示 | DEPART, APPROACH |
| CINE_SIDE | 併走ドローン風。車両側面をゆっくり追い越す | CRUISE |
| CINE_FRONT | 線路脇の固定カメラ。列車が迫って通過（ドップラー音） | CRUISE |
| CINE_AERIAL | 斜め上空から風景ごと俯瞰、ゆっくりパン | CRUISE |
| CINE_TAIL | 最後尾から線路が流れ去る画 | CRUISE |
| SIDE_STOP | ホーム側の低い横アングル。先頭車と停止線を同一フレームに | FINAL, STOPPED |
| PLATFORM | ホーム全景（結果演出の背景） | STATION_RESULT, DOORS |

**遷移の演出規則（ハードカット禁止の原則）**:
- 状態をまたぐ遷移 = **フェード**（黒0.35s。黒帯に駅名標風の白ラインを走らせる専用トランジション）
  - CRUISE→APPROACH のみ**警告音＋フェード**（「運転再開」の合図として重要。600m標識が窓外を過ぎる）
- CRUISE内のシネマティックカット切替 = **直カット**（映画の文法。1ショット4〜7s、シード乱数で順序を変える）
- FINAL への切替のみ**スウィッシュパン**（COCKPIT→SIDE_STOPへカメラが横に高速スイング、0.4s、風切り音）。停止直前の緊張を切らさないため黒フェードは使わない
- カメラ揺れ: COCKPITは速度に応じた微振動＋ジョイント通過でガタン、CINE系は手持ち風の微揺れ

## 3. 運転物理（1次元・決定論）

- 線路に沿った1次元: 位置x、速度v。`update(dtMs)` 固定サブステップ
- 自動運転（DEPART/CRUISE）: ノッチP3相当で加速→巡航75km/h→APPROACH開始
- **開始条件は毎回ランダム**（シード付き）: 残距離 420〜520m・速度 62〜75km/h・勾配 -10〜+10‰ を駅ごとに変える。固定条件だと「特定ノッチを握って放置」が最適解になってしまうため。残距離とKEの比が毎回変わることで、プレイヤーは残距離表示と速度計から減速度を組み立てる必要がある（これが本作の腕）
- ブレーキ: ノッチ N, B1〜B8, EB
  - 減速度: B1=0.25 → B8=1.10 m/s²（等間隔ではなくB5以降を厚く）、EB=1.60
  - **空気ブレーキの応答遅れ**: 指令から実効まで約0.6sの一次遅れ（ここが腕の見せ所）
  - 緩め方向も同様に遅れる。EB使用は即減点フラグ
- 勾配: 区間ごとに±10‰程度の下り/上りを1箇所（残距離掲示だけでは止まれない変化要素）
- 停止時ジャーク判定: 停止瞬間の減速度が0.5m/s²超なら「ガクン停車」（乗客よろけSE＋減点）。理想は停止直前にB1〜B2へ抜く操作

## 4. スコアリング

駅ごとに100点満点:
- 停止誤差: |誤差| ≤30cm=100「ピタリ!!」/ ≤1m=80「上出来」/ ≤2m=55「まずまず」/ ≤5m=30 / それ以上・CREEP使用=10
- 減点: ガクン停車 -15、EB使用 -30、OVERRUN=0点固定（10m超過時）
- ボーナス: ノッチ操作回数が少ない（≤6回）+5「滑らか運転」
- 総合: 4駅合計から称号（380+=名人運転士 / 300+=ベテラン / 200+=一人前 / 100+=見習い / 未満=研修やり直し）。ベストをlocalStorage保存

## 5. HUD（作り込み指定）

- **速度計**: 左下に大型の円形アナログメーター。黒文字盤・白目盛・赤針、中央にデジタル併記（7セグ風）。針は物理値に追従して滑らかに
- **ブレーキレバー**: 右端に縦型レバー。ノッチ位置がカチッカチッと段階移動（ラベル N/B1..B8/EB）。操作時にレバー自体が傾く
- **残距離表示**: APPROACH中は運転台上部に電光掲示風オレンジ7セグ「のこり 428m」。FINALでは画面中央下に**大型カウンタ「3.42m」**（小数2桁、1m未満でcm表示に切替、色が白→黄→赤）
- **次駅案内**: 上部に車内LED風の黒帯「次は たばがわ 田場川」（DEPART時にスクロールイン）
- **路線図**: RUN_INTROと結果画面に、駅丸＋現在位置の路線図
- FINALの誤差確定瞬間: 停止線と先頭車の間に**寸法線＋数値がドンと出る**（測量図風の演出）

## 6. 3D世界

- 線路: 区間ごとのスプライン（直線基調＋緩S字1つ＋トンネルor鉄橋1つ）。バラスト・2本レール・枕木・架線柱（等間隔リピート）
- 車両: 2両編成の気動車風ローポリ（前面貫通扉・白+朱色の国鉄風帯・台車・床下機器）。窓は発光面（夕方区間で映える）
- 駅: 単式ホーム・上屋・駅名標・**停止位置目標（白線＋「2」札）**・ベンチ・自販機の光
- 沿線: 区間テーマ = ①田園（水田の反射）②住宅地（家々・踏切）③山間（トンネル・鉄橋・川）④海沿い夕焼け（最終駅、逆光でクライマックス）
- 空: 時間経過グラデ（昼→午後→夕→マジックアワー）。遠景山シルエット
- 踏切: 区間2に1箇所。CINE_FRONTカメラと組み合わせてカンカン＋通過を演出

## 7. SE（WebAudio自前合成・日本の鉄道らしさ）

- **VVVFインバータ風加速音**: 発車時に音階が段階的に上がる磁励音（矩形波+フィルタで近似）※気動車モチーフだが音の気持ちよさ優先でVVVF採用
- **ジョイント音**: 「ガタンゴトン…ガタンゴトン」速度連動の2連打パルス（レール25m周期、台車2つぶん）
- ブレーキ: 緩解の「プシューッ」、込めの「シュー」、停止直前のキィーッ（微少）
- 発車ベル「ジリリリ」/ ドアチャイム「ピンポーン」×2 / ドア開閉「プシュー・ガラガラ」
- 踏切「カンカンカン」（接近でドップラー）、警笛（タッチ/Hキーでおまけ搭載）
- 走行風・トンネル残響（トンネル内はジョイント音にリバーブ）
- BGMなし（環境音で構成するのが電GO文化。TITLE/RESULTにだけ短いジングル）

## 8. 操作

- PC: ↓/S=ブレーキ込め(1段) ↑/W=緩め(1段)、1〜9=直接ノッチ、E=EB、H=警笛、Space=CRUISEスキップ
- タッチ: 右端のレバーを上下ドラッグ（ノッチにスナップ）、画面タップ=CRUISEスキップ、警笛ボタン
- 横持ち推奨（HELLBREAKと同じ回転プロンプト方式）

## 9. デバッグAPI

- getState(): phase/x/v/notch/残距離/誤差/スコア/駅index/ショット名
- step(ms) / brake(n) / setSpeed(kmh) / setDist(m) / skipTo(phase) / finishStation(誤差m注入) / result()
- validate(): 各区間で「残600m・75km/hからB5一定で停止線±0内に止まれる解が存在する」ことを物理式で起動時検証（勾配込み）
- __renderOnce / ?debug=1 / ?seed=N

## 10. 実装フェーズ分割（Codex発注単位）

- **Phase 1 — コア**: 状態機械・1次元物理（遅れ込みブレーキ）・スコアリング・HUD一式（メーター/レバー/残距離/LED）・仮ジオメトリ（灰色の線路と箱車両）で APPROACH→FINAL→STOPPED→RESULT が1区間遊べる。validate実装。カメラはCOCKPITとSIDE_STOPの2つ＋フェードのみ
- **Phase 2 — 世界とシネマ**: 4区間の3D世界（スプライン線路・車両モデル・駅・沿線4テーマ・空の時間変化）、DEPART/CRUISE/DOORSの自動運転、CINE_*カメラ4種と直カット/スウィッシュパン等の遷移演出、RUN_INTRO/FINAL_RESULT
- **Phase 3 — 音と磨き**: SE一式（VVVF/ジョイント/ドア/ベル/踏切/風）、FINALの寸法線演出・結果演出の強化、ジャーク判定の調整、モバイル最適化・横持ちプロンプト、バランス調整

各フェーズ完了ごとにオーケストレーターがレビュー・実機検証してから次へ進む。
