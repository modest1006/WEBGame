# BUBBLE BLASTER EX

壁で反射させて狙い撃ち、同色3つ以上を繋げて消す「パズルボブル」系オリジナルパズル。2000年代アーケード筐体の懐かしさを、3D（光沢バブル・回転する背景）と2D（メタリックベゼル・カットイン演出）の組み合わせで表現。

## 遊び方

- 下部中央の砲台からバブルを発射し、同色3つ以上を繋げて消す
- 弾は左右の壁で反射する。天井または既存バブルに触れた位置の最寄りグリッドセルにスナップする
- 消滅後、天井から切り離された孤立クラスタはまとめて落下し、落下ボーナス点が入る
- 発射数回ごとに天井が1段下がる。バブルが画面下部のデッドラインを超えるとゲームオーバー
- 全消し（盤面クリア）でステージクリア。残り弾数ボーナス付きで次ステージへ。全8ステージ、後半ほど高密度・多色
- NEXT表示のバブル色は盤面に残存する色からのみ抽選（詰み色を配らない）
- コンボ: 1手のポップ＋落下合計が多いほど倍率が上がり、カットイン演出と連動

## 操作方法

**PC**
- マウス移動で照準、クリックで発射
- ←→キーで照準の微調整、Spaceで発射
- R: リスタート / M: ミュート / P: ポーズ

**タッチ**
- ドラッグで照準（指の位置に追従）、離すと発射
- 砲台付近には誤爆防止のデッドゾーンあり

## 起動方法

- `index.html` をダブルクリックして直接開く（`file://` 対応、ビルド不要）
- ローカルサーバー例: `python -m http.server 8135 --directory games/bubbleex`

## デバッグAPI（`window.__game`）

- `getState()` — 盤面セル・現在弾・NEXT・スコア・ステージ・天井までの残弾・状態などのスナップショット
- `dump()` — 盤面のASCII表現（`R`赤 `B`青 `G`緑 `Y`黄 `P`紫 `O`橙）
- `step(ms)` — 決定論的に時間を進める
- `aim(deg)` — 照準角度を設定（±80度）
- `fire()` — 発射
- `setBubble(color)` — 現在弾の色を強制設定
- `setStage(n)` — 指定ステージ（1〜8）を読み込み
- `clearBoard()` — 盤面を全消去
- `setCell(row, col, color)` — 指定セルにバブルを直接配置（`color=null`で削除。検証用）
- `win()` — 即座にステージクリア扱いにする
- `lose()` — 即座にゲームオーバー扱いにする
- `shootAt(row, col)` — 指定グリッドセルに向けて角度を逆算し、物理シミュレーションを進めて即時着弾させる（障害物がある場合は物理的に到達可能な位置に着弾する点に注意）
- `validate()` — 全8ステージについて「配置色が全て3個以上存在する」「デッドラインより下に初期配置がない」を検証
- `restart()` — スコアをリセットしてステージ1からやり直す
- `togglePause()` — ポーズ切り替え

`?debug=1` でデバッグオーバーレイ表示、`?seed=N` で乱数シード固定（NEXT抽選が再現可能に）。

## 実装メモ

- 盤面は六角オフセットグリッド（偶数行8列・奇数行7列の互い違い配置）
- `js/game.js` はDOM/Three.js非依存の決定論ロジック（衝突判定・同色連結のフラッドフィル・天井連結BFSによる孤立クラスタ落下判定）
- `js/renderer.js` のみThree.jsを使用し、光沢バブル（MeshPhysicalMaterial + clearcoat）・回転する背景オブジェクト・砲台3Dモデル（装填中の現在弾＋待機NEXT弾・装填スピンアニメ付き）・最前面描画の点線照準ガイドを描画
- 消滅演出は「ポップコーン連鎖」: 着弾セルからBFS距離順に35ms差で1個ずつ膨張→光沢シャード＋フラッシュリング＋星型スパークで飛散（5個以上同時消しでシャード増量＋画面シェイク＋虹色スパーク）。孤立クラスタは重力1800px/s²の物理落下アニメ後に画面外で消滅
- `window.__renderOnce(dtMs)` でrAFなしに1フレーム描画＋エフェクトをdtMsぶん進行（ヘッドレス検証用）。`window.__renderer` でレンダラ内部を参照可能
- 2D UI（筐体ベゼル・ロゴ・HUD・カットイン）はCSS/DOMで実装
- 音はWebAudioのオシレータ合成（外部音源ファイルなし）
